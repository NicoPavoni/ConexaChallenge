Plan de Automatización

Estrategia del plan de automatización

  • Selección de herramienta y/o framework de automatización:
    • • Python: Se seleccionó Python como lenguaje de programación debido a su sintaxis clara, versatilidad y porque es uno de los lenguajes mas sencillos y con curva de aprendizaje buena.
    • • Selenium: Selenium se eligió como la herramienta principal para la automatización de pruebas de interfaz de usuario (UI).
    • • Pytest: Pytest se adoptó como el framework de pruebas debido a su simplicidad, flexibilidad y características de informes y registro.
    • • Page Object Model (POM): Se implementó el patrón POM para mejorar la modularidad y mantenibilidad del código.
  • ¿Por qué la elección de la herramienta y/o framework de automatización?
    • • Python: La elección de Python se basa en su amplia comunidad de desarrollo, extensa biblioteca estándar y su uso generalizado en el campo de la automatización y desarrollo de pruebas.
    • • Selenium: Selenium es una herramienta líder en la industria para la automatización de pruebas de UI y cuenta con el fuerte de tener mucho soporte.
    • • Pytest: Pytest se seleccionó por su simplicidad y potencia.
    • • Page Object Model (POM): El patrón POM se adoptó para mejorar la reutilización del código, reducir la duplicación y mejorar la mantenibilidad.

Buenas prácticas para la organización y ejecución de las pruebas.

  • Buenas prácticas :
    • • Utilización del Patrón Page Object (POM): Organizar el código utilizando el patrón Page Object, que separa la lógica de la interfaz de usuario en clases reutilizables. Cada página o componente de la interfaz debe tener su propia clase Page Object.
    • • Dividir los Tests en Archivos y Carpetas Lógicas: Estructurar las pruebas en archivos y carpetas basadas en la funcionalidad o el módulo que están probando. Esto facilita la navegación y mantenimiento del código.
    • • Informes Detallados: La configuración de pruebas para generar informes detallados por ejemplo, como en este caso, informes HTML con pytest. Esto facilita la identificación y corrección de problemas cuando se producen errores.
    • • Code Review: Fomentar el code review entre los miembros del equipo. Esta practica permite identificar problemas antes de que se fusionen los cambios.
